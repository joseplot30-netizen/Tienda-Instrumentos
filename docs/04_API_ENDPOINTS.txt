# API ENDPOINTS - Documentación Completa de Rutas

## 🔐 Autenticación

### Sesión de Usuario

Todas las rutas protegidas requieren autenticación. La sesión se inicia mediante `/inicio_sesion` y se valida automáticamente.

**Variables de Sesión disponibles:**
```python
session['user_id']    # ID del usuario (int)
session['username']   # Nombre de usuario (str)
session['is_admin']   # Flag de administrador (bool)
session['role']       # 'admin' o 'user' (str)
```

## 📱 Rutas Públicas

### GET `/`
**Descripción**: Página de inicio  
**Autenticación**: No requerida  
**Respuesta**: HTML (página de inicio)

```
GET / HTTP/1.1
→ 200 OK
→ Render 'pages/index.html'
```

---

### GET `/guitarras`, `/bajos`, `/baterias`, `/otros`
**Descripción**: Catálogo por categoría  
**Parámetros**: Ninguno  
**Autenticación**: No requerida  
**Respuesta**: HTML con productos filtrados

### POST `/inicio_sesion`
**Descripción**: Procesar autenticación de usuario  
**Autenticación**: No requerida  

**Parámetros (form-data):**
```
username: string (requerido)
password: string (requerido)
```

**Respuesta exitosa:**
```
POST /inicio_sesion HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=juanperez&password=secreto123

→ 302 Redirect
→ Location: /
→ session['user_id'] = 1
→ session['username'] = 'juanperez'
→ session['is_admin'] = false
→ Flash: '¡Bienvenido juanperez!'
```

**Respuesta fallida:**
```
→ 200 OK
→ Render 'pages/INICIO_SESION.html'
→ Flash: 'Usuario o contraseña incorrecta'
```

### POST `/registro`
**Descripción**: Crear nueva cuenta de usuario  
**Autenticación**: No requerida  

**Parámetros (form-data):**
```
username: string (requerido, 3-50 chars)
email: string (requerido, email válido)
password: string (requerido, 6+ chars)
password2: string (requerido, debe coincidir)
```

**Validaciones:**
- ✓ Campos no vacíos
- ✓ Passwords coinciden
- ✓ Username único en BD
- ✓ Email único en BD

**Respuesta exitosa:**
```
POST /registro HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=nuevo_usuario&email=user@email.com&password=123456&password2=123456

→ 302 Redirect
→ Location: /inicio_sesion
→ Flash: 'Usuario creado correctamente. Inicia sesión.'
→ INSERT en tabla users
```

**Respuesta fallida:**
```
→ 200 OK
→ Render formulario
→ Flash: 'El usuario ya existe' | 'Las contraseñas no coinciden' | etc.
```

## 🔒 Rutas Protegidas (Requieren Admin)

### GET `/admin/users`
**Descripción**: Listar todos los usuarios  
**Autenticación**: Requerida (admin)  
**Respuesta**: HTML con tabla de usuarios

```
GET /admin/users HTTP/1.1
Cookie: session=...

→ 200 OK
→ Query: SELECT * FROM users
→ Render 'admin/admin_users.html'
→ Incluye búsqueda y filtros en tiempo real
```

**Características:**
- Búsqueda por username, email, ID
- Filtros: Todos, Administradores, Usuarios
- Contadores por rol
- Acciones: Editar, Eliminar

---

### GET/POST `/admin/users/create`
**Descripción**: Crear nuevo usuario  
**Autenticación**: Requerida (admin)  

**GET - Mostrar formulario:**
```
GET /admin/users/create HTTP/1.1
→ 200 OK
→ Render form 'admin/admin_create_user.html'
```

**POST - Procesar creación:**
```
POST /admin/users/create HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=admin01&email=admin@example.com&password=secure&is_admin=1

→ 302 Redirect
→ Location: /admin/users
→ Flash: 'Usuario creado correctamente'
→ INSERT en tabla users
```

---

### GET/POST `/admin/users/edit/<user_id>`
**Descripción**: Editar usuario existente  
**Autenticación**: Requerida (admin)  
**Parámetros URL**: `user_id` (int)

**GET - Mostrar formulario con datos actuales:**
```
GET /admin/users/edit/5 HTTP/1.1
→ 200 OK
→ Query: SELECT * FROM users WHERE id=5
→ Render form con valores precargados
```

**POST - Procesar actualización:**
```
POST /admin/users/edit/5 HTTP/1.1

username=new_name&email=new@email.com&is_admin=0

→ 302 Redirect
→ Location: /admin/users
→ Flash: 'Usuario actualizado'
→ UPDATE users SET ... WHERE id=5
```

---

### POST `/admin/users/delete/<user_id>`
**Descripción**: Eliminar usuario  
**Autenticación**: Requerida (admin)  
**Parámetros URL**: `user_id` (int)


## 📄 Códigos de Estado HTTP

| Código | Significado | Ejemplo |
|--------|-----------|---------|
| 200 | OK | GET exitoso |
| 302 | Found (Redirect) | POST exitoso con redireccionamiento |
| 400 | Bad Request | Parámetros faltantes |
| 401 | Unauthorized | No autenticado (redirige a login) |
| 403 | Forbidden | Autenticado pero sin permisos admin |
| 404 | Not Found | Recurso no existe |
| 500 | Server Error | Error en el servidor |

## 🛡️ Manejo de Errores

### Flash Messages
Los errores y mensajes se comunican al usuario mediante flash messages:

```python
# Éxito
flash('Operación completada', 'success')

# Error
flash('Hubo un error', 'danger')

# Información
flash('Por favor complete todos los campos', 'info')

# Advertencia
flash('Debes iniciar sesión', 'warning')
```

### Validación de Formularios

**Cliente (JavaScript):**
- Campos no vacíos
- Formatos válidos (email, etc)

**Servidor (Flask):**
- Validación adicional
- Verificación de unicidad
- Control de permisos
